jenkins远程自动打包、多渠道打包

发表于 2018-09-23 | 分类于 Android

Jenkins功能强大，可以使用它进行android项目的远程打包、多渠道打包工作，并且可以自动提交标签发送邮件。
下面就介绍一下在实际项目中使用到的Jenkins自动打包教程。

Jenkins简介

Jenkins是目前开源的应用中最广泛的持续集成工具，支持CI, CD。2.0开始支持Pipeline，使用基于Groovy的DSL，可以实现单个任务难以完成的复杂发布流程。其任何发布流程都可以表述为一段Groovy脚本，并且Jenkins支持从代码库直接读取脚本，从而实现了实时或轮询打包的功能。

Jenkins功能强大，在实际使用上可以使用它进行Android项目的远程打包、多渠道打包、自定义配置项打包等工作，并且可以自动发送附件邮件等功能，因为其支持shell脚本命令，所以也可以通过shell命令进行多种需求的操作。
下面就介绍一下在实际项目中使用到的Jenkins自动打包教程和相关功能拓展。

1，下载Jenkins

Jenkins官网下载地址： https://jenkins.io/download/
根据自己的操作系统选择对应版本

2，安装Jenkins

点击下一步按照流程按照即可，不做赘述。

3，初始化Jenkins

Jenkins安装成功后可以在本地访问 http://localhost:8080

第一次初始化如图：

解锁：安装提示路径打开文件，复制密码，点击继续

按照弹框提示，找到该initialAdminPassword，填写Jenkins生成的key：

插件选择

推荐选择左边，安装官方推荐的插件，也可以选择右边只安装自己需要的插件。

创建用户：

这个用户是下次访问 http://localhost:8080 时需要登陆的用户名和密码

初始化完成，进入使用界面

4，配置Jenkins

进入Jenkins不需要先新建项目，首先把需要的环境配置好，具体需要配置的地方如下。
①系统设置
点击“系统管理”

点击“系统设置”

找到“全局属性”，勾选“Environment variables”，然后配置环境变量（填写自己电脑上的对应的路径）

设置局域网访问地址：

设置通知邮件地址和内容格式

每封邮件都默认的内容

②全局安全配置Configure Global Security
默认的设置不变
点击“项目矩阵授权策略”可以对自己添加的用户进行权限设置

③全局工具配置Global Tool Configuration
JDK 和 Git 保持默认设置
Gradle配置：
点击“新增Gradle”

填写自定义名称，然后选择自己需要的Gradle版本（可以新增多个版本的Gradle）

5，创建打包项目

新建任务

选择任务类型

勾选参数化构建过程

参数化构建可以在编译前选择自己需要配置的选项，再利用这些选项实现不同类型的编译任务。
可选参数如下

源码管理，勾选git

在URL处填写项目的git地址，下面一栏需要选择代码地址对于的账号。

点击Add

使用Username with password 时，Username一栏填写代码地址的账户名，Password填写密码，其他默认。
在添加后返回，在Credentials一栏选择刚才添加的账号，如果提示红色的报警，说明没有验证通过。此时可以用另外一种方式再添加。
选择ssh验证，并在key的地方填写自己的私钥，添加后再返回选择自己的账号就OK了。

在Branches to build处填写要打包的分支

在构建里选中 Invoke Gradle，然后选择自己在系统设置里配置的gradle版本，在Tasks栏下填写自己要编译的task名称（这里是先执行clean，再执行编译任务）

保存后返回，选择参数化构建

选择要构建的任务，点击开始构建

可以点击左方的小点看编译的日志

Jenkins会先在git仓库拉下填写分支的最新代码到本地工作目录，然后执行代码中的gradle命令进行编译。当日志显示BUILD SUCCESSFUL则编译成功。

可以到工作空间下找到自己编译好的apk

至此就完成了一个最基本的自定义编译任务的Jenkins远程打包功能。

6，构建拓展

Jenkins支持shell脚本命令，所以可以通过shell去添加很多功能。

在构建的栏目下选择添加构建步骤,选择执行shell

在shell里编写命令

通过这一点，可以在执行代码build前后去处理一些初始化，比如编译前拿到自定义配置项去替换代码里的内容，编译后通过git提交tag、通过curl去触发某些接口，可以无限拓展。

举例：打包成功后可以通过shell命令向钉钉群里发送消息：
curl -X "POST" "https://oapi.dingtalk.com/robot/send?access_token=$DingDingId" \ -H "Content-Type: application/json" \ -d ' {"msgtype": "text", "text": { "content": "Android '$BUILD_BRANCH' 分支的'$BUILD_TYPE'测试包已打好，请下载测试：https://fir.im/androidXXX" } }'

7，插件拓展

Jenkins的方便之处还在于它有很多插件可以使用。
比如可以下载fir的Jenkins插件，通过配置去把apk自动上传到fir上。

总结

以上就是Jenkins在Android远程打包上的使用过程，主要写了环境的搭建和重要环节的配置。实际使用时需要配合这项目实际需求已经代码的构建情况去搭建一个最方便快捷的远程编译平台。
